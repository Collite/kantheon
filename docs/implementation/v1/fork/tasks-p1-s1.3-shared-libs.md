> **Package-root convention:** the ai-platform libs use Kotlin package roots like `shared.otel.*`, `shared.logging.*`, `shared.ktor.*`, `shared.translator.*`, `shared.libs.db.common.*`, `org.fuzzy.common.*` — **not** `org.tatrman.kantheon.*`. Per the task doc ("Imports in Kotlin sources don't change (same packages inside the libs — only coordinates moved)"), the fork preserves these package roots as-is. The standard kantheon module layout (§4 of AGENTS.md) still applies to **first-party services** (agents/, tools/, services/); the **forked libs are a deliberate exception** because changing package roots would be a code-level fork churn of dozens of files per lib with no functional benefit (the libs never appear in any public kantheon API surface). Recorded here for reviewers and for the eventual Phase 4 docs sweep.
