## Grounding (codebase as of 2026-06-23)

- **BFF `/v1/action`** (`api/ActionRoutes.kt`) handles `edit_resend` and streams a `NOT_IMPLEMENTED` error for every other `kind` (`sort`/`filter`/`paginate`/`select_row`/`chip_invocation`/`reask_agent`). `api/ActionDtos.kt` has `TypedActionRequestDto{sessionId, bubbleId?, action{kind, payloadJson}}` + `EditResendPayload`. **This stage fills the `else` branch** with per-kind dispatch.
- **Data-shaping mechanism (resolved 2026-06-23, Bora).** `sort`/`filter`/`paginate` are applied **BFF-side on the cached rows** of the target bubble — the rows are already in `iris_turns.envelope_json` (`content_json`) for the producing turn, so no agent round-trip is needed for re-ordering / sub-setting / paging a result already in hand. `select_row` + drilldown, and any action whose result needs data **not** in the cached envelope (a refetch — e.g. a filter that widens beyond the cached page, or a drilldown to a different pattern), re-issue against the producing agent. Transitionally the producing agent is new-golem `/v2`: a refetch is sent as a re-issued `/v2/chat` against the same `thread_id` with the action encoded in the question/args (contracts §5 adapter mapping; `GolemV2Client` gains a `reissueAction(threadId, bubbleId, action)` verb that wraps `/v2/chat/stream`). All shaping/refetch emits a **replacing** envelope with the **same `bubble_id`** (contracts §2.4).
- **EntityContext**: `iris_sessions.entity_context` JSONB exists; `FormatEnvelope.entity_context` (repeated `EntityContextSnapshot`) is parsed by the FE today. The BFF does not yet read response `entity_context` back into the session or compare `applied_context` (PD-4) — that wiring is here.
- **FE**: `services/typedAction.ts` is the throwing Phase-3 stub (Stage 2.3 implemented `editResend` against `/v1/action`; the rest still degrade to plain `irisStream.turn`). `stores/chatStore.ts` holds per-bubble `displayState`; `stores/chipStore.ts` holds chips; `ChatBubble.vue` renders bubbles. Agent badge / re-ask picker / Investigate affordance are all new FE surfaces.
- **capabilities-client** supplies display names + role-filtered routable agents for the re-ask picker and the agent badge.
