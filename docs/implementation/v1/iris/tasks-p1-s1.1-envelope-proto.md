## Decisions locked at stage start

- **TS codegen tool = `ts-proto`** (Bora-confirmed 2026-06-17; plan §8 open question closed). Plain TS interfaces + `fromJSON`/`toJSON` — near drop-in for the hand-written `agents-fe/src/types/envelope.ts` it replaces in Phase 2, minimising FE-adoption churn.
- **`ts-proto` option `snakeToCamel=keys`** — idiomatic **camelCase** TS message keys (`bubbleId`) while keeping **snake_case** JSON on the wire (`fromJSON` reads `bubble_id`, `toJSON` emits it). This is the golden-sample-compatible setting: the default `keys_json` would make `fromJSON` expect camelCase JSON and silently miss every v2 snake_case field.
- **Codegen driver = `buf` (`@bufbuild/buf` npm devDep) + local `ts-proto` plugin** — self-contained, **no system `protoc`** (this box has none; the Gradle protobuf plugin owns the KT/PY protoc). `buf generate --path` scopes TS output to envelope/v1 + iris/v1 + common/v1 only; the rest of the proto tree (theseus, argos, …) is **not** emitted into the FE bundle.
- **Golden samples = synthesised from the v2 source-of-truth shapes** (`ai-platform/frontends/agents-fe/src/types/envelope.ts` + `agents/golem/src/api/v2/models.py`), **flagged for live-capture replacement** (plan pre-flight: "recorded /v2 SSE + envelope fixtures from a live new-golem session"). No live new-golem is reachable from this box; the fixtures exercise the real v2 quirks visible in source (flattened format details, `content` as opaque JSON, chips, clarification, drilldowns). CI gate is real once live captures land — tracked as a fixture-refresh follow-up.
