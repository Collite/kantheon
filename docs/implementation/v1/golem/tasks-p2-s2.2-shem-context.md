## Decisions taken at stage start (2026-06-18)

- **Ariadne client is promoted to a shared lib** (`shared/libs/kotlin/ariadne-client`), and `tools/ariadne-mcp` is migrated to consume it (Bora, 2026-06-18). Golem's `PackageContext` + `PromptStore` consume the same client; Pythia will too. Rationale: avoid a 2nd/3rd copy of the gRPC client; consolidate-first. The client needs only `:shared:proto` (it generates the grpc + grpckt stubs) + grpc-netty + coroutines — **not** `:services:ariadne`.
- **`ShemManifest` = `AgentCapability` with `agent_kind == DOMAIN_QA`** (no separate proto). Golem's `ShemLoader` parses its own Shem YAML (golem-local, distinct from capabilities-mcp's server-side `ManifestYamlLoader`, to avoid an `agents/golem → tools/capabilities-mcp` dep). The Golem loader **does** map `visibility_roles` (the capabilities-mcp loader does not yet — admission needs it).
- **`PackageContext` / `PromptStore` cache on the Ariadne hashes** — `ModelBundle.package_versions[].content_hash` and `GetPromptsResponse.tree_hash` — using a Caffeine TTL cache (the Themis `ResolverCache` idiom).
