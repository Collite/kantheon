> **Arc.** Golem Phase 3 (conversational surface). **Branch.** `feat/golem-p3-s3.1-format-pipeline`.
> **Companions.** [`plan.md`](./plan.md) §5 Stage 3.1 (+ §10 Δ2/Δ5), [`../../../architecture/golem/architecture.md`](../../../architecture/golem/architecture.md) §3 (module map) / §4 (graph), [`../../../architecture/golem/contracts.md`](../../../architecture/golem/contracts.md) §3 (SSE) / §4 (golem_turns), [`../../../architecture/iris/contracts.md`](../../../architecture/iris/contracts.md) §1.1 (envelope/v1 + InvestigateChip), [`../../../architecture/kantheon-security.md`](../../../architecture/kantheon-security.md) (PD-9 provenance). `shared/libs/kotlin/envelope-render` (Phase 1), `shared/libs/ts/envelope-ts` (Iris Stage 1.1).

>
> **⚠ Parity baseline (2026-06-24, plan.md §10 Δ2/Δ5).** This stage is refreshed against the current ai-platform Golem (`nodes_v2/format.py`, `format/kind_inference.py`, `chips/*`, `drilldowns.py`, `api/v2_routes.py`): chips have **4 sources** (`static | heuristic | pattern_derived | llm_topup`); the heuristic chips are hard-coded `ucetnictvi` column literals (package-coupling debt — flag, don't generalise); `CurrentView` carries **`total_rows`**, `FormatEnvelope` carries **`update_tab_id`** + **`losing_plan_summary`**; **kind-inference** precedence is `pattern result_kind_hint > amend-on-compare > table` (chart-on-compare gated `GOLEM_CHART_ON_COMPARE_ENABLED`); the SSE event-name set is **fixed** and there is **no live-log `/log` stream** (removed upstream Phase 8); **outbound OTel W3C `traceparent`** propagation is now expected on every outbound call.
