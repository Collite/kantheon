# Agents Frontend (agents-fe)

## Overview

A Vue 3 + TypeScript frontend application that exposes communication with agents. It provides interfaces for interacting with the ERP Agent, LLM Gateway, and MCP Inspector.

## Tech Stack

- **Framework:** Vue 3.5 + TypeScript
- **Build Tool:** Vite 7
- **State Management:** Pinia
- **Routing:** Vue Router 4
- **Styling:** Tailwind CSS 3
- **HTTP Client:** Axios
- **Authentication:** Keycloak (via keycloak-js)
- **Telemetry:** OpenTelemetry (OTLP)
- **MCP Client:** @modelcontextprotocol/sdk

## Configuration

### Environment Variables

All configuration is injected at runtime via `window.APP_CONFIG` (populated by `env.js` in Docker).

| Variable | Description | Default |
|----------|-------------|---------|
| `VITE_AUTH_ENABLED` | Enable/disable Keycloak auth | `true` |
| `VITE_KEYCLOAK_URL` | Keycloak server URL | - |
| `VITE_KEYCLOAK_REALM` | Keycloak realm | - |
| `VITE_KEYCLOAK_CLIENT_ID` | Keycloak client ID | `agents-fe` |
| `VITE_TELEMETRY_ENABLED` | Enable OpenTelemetry | `false` |
| `VITE_OTEL_EXPORTER_OTLP_HOST` | OTEL collector host | `localhost` |
| `VITE_LLM_GTW_SERVER` | LLM Gateway server host | `localhost` |
| `VITE_LLM_GTW_SERVER_PORT` | LLM Gateway server port | `7101` |
| `VITE_ERP_AGENT_SERVER` | ERP Agent server host | `agent.erp.aip01.dfpartner.cz` |
| `VITE_ERP_AGENT_SERVER_PORT` | ERP Agent server port | - |
| `VITE_MCP_ERP_SERVER` | ERP MCP server host | `localhost` |
| `VITE_MCP_ERP_SERVER_PORT` | ERP MCP server port | `7151` |
| `VITE_MCP_FUZZY_SERVER` | Fuzzy MCP server host | `localhost` |
| `VITE_MCP_FUZZY_SERVER_PORT` | Fuzzy MCP server port | `7152` |
| `VITE_AGENTS_FE_PROTOCOL` | HTTP protocol (http/https) | `http` |

## Features

### 1. ERP Agent (AgentView)

Chat interface for communicating with the ERP Agent.

**Capabilities:**
- Multi-language support (EN, DE, CS, SK, HU)
- Streaming response display
- Markdown rendering in chat bubbles
- Language preference persisted in localStorage

**Key Files:**
- `src/views/AgentView.vue` - Main chat interface
- `src/services/agentService.ts` - Agent API client
- `src/components/chat/ChatBubble.vue` - Message display with markdown
- `src/components/chat/ThinkingIndicator.vue` - Loading animation

### 2. MCP Inspector (InspectorView)

Tool explorer for MCP (Model Context Protocol) servers.

**Capabilities:**
- Connect to ERP MCP or Fuzzy MCP servers
- List available tools
- Search/filter tools by name or description
- Execute tools with JSON arguments
- View execution results in log viewer

**Key Files:**
- `src/views/InspectorView.vue` - Main inspector interface
- `src/services/mcpClient.ts` - MCP SDK client wrapper
- `src/components/inspector/ConnectionPanel.vue` - Server connection UI
- `src/components/inspector/ToolList.vue` - Tool browser with search
- `src/components/inspector/ToolExecutor.vue` - Tool execution form

### 3. LLM Gateway (LlmGatewayView)

Interface for testing the LLM Gateway API.

**Capabilities:**
- Simple mode: model tag + user query
- Advanced mode: system prompt, temperature, max tokens, reasoning effort
- Conversation context support
- JSON response display

**Key Files:**
- `src/views/LlmGatewayView.vue` - Gateway interface
- `src/services/llmGatewayService.ts` - LLM Gateway API client

## Authentication

Keycloak integration with the following flow:
1. App initializes Keycloak via `initKeycloak()`
2. If `VITE_AUTH_ENABLED=false`, bypasses auth with dev user
3. Auth state managed in Pinia store (`stores/auth.ts`)
4. Protected routes redirect to Keycloak login if not authenticated

## Telemetry

OpenTelemetry integration for observability:
- **Traces:** WebTracerProvider with OTLP exporter
- **Logs:** LoggerProvider with OTLPLogExporter
- **Metrics:** MeterProvider with OTLPMetricExporter

To enable, set `VITE_TELEMETRY_ENABLED=true` and configure OTEL collector endpoints.

## Deployment

### Docker

```bash
# Build image
docker build -t agents-fe:latest .

# Run container
docker run -p 7011:7011 \
  -e VITE_KEYCLOAK_URL=http://keycloak:8080 \
  -e VITE_KEYCLOAK_REALM=df-test \
  -e VITE_KEYCLOAK_CLIENT_ID=agents-fe \
  agents-fe:latest
```

### Nginx Configuration

The application runs behind nginx which handles:
- Static file serving from `/agents/fe/`
- Reverse proxy to backend services
- WebSocket/SSE support for streaming endpoints

## Known Limitations

- Agent language preference is stored client-side only
- No retry logic on failed API calls
- MCP tool schema validation is basic
- No built-in rate limiting feedback
